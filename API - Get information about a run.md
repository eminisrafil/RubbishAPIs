# Rubbish API Documentation - Get information about a run

## GET /runs/:teamID/movement/:runID


This endpoint retrieves detailed movement information for a specific run executed within the Rubbish application. Each run consists of an array of objects, where each object represents a location data point. These data points track the user's movement since the last recorded location and include information such as user steps, distance, speed, pace, and changes in the litter count.

### Headers

- `Authorization`: Bearer token for authentication

### URL Parameters

- `teamID`: The ID of the team for which the runs are to be retrieved.
- `runId`: The ID of the run for which the i retrieved.

### Response

Returns an array of objects, each representing a user's movement in the run.

| Key                 | Type    | Description |
| ------------------- | ------- | ----------- |
| debugIndex          | Number  | Debug index of the run |
| distance            | Number  | Distance covered in the run |
| duration            | Number  | Duration of the run |
| isTemporaryModel    | Boolean | Indicates if the model is temporary |
| isUploadedToFireStore| Boolean| Indicates if the data is uploaded to Firestore |
| lat                 | Number  | Latitude of the run location |
| litterDelta         | Number  | Change in litter count |
| litterPer100Steps   | Number  | Litter count per 100 steps |
| litterPer10Meters   | Number  | Litter count per 10 meters |
| long                | Number  | Longitude of the run location |
| modelVersion        | Number  | Version of the model |
| movementModeID      | String  | ID of the movement mode |
| pace                | Number  | Pace of the run |
| rubbishRunStoryID   | String  | ID of the rubbish run story |
| speed               | Number  | Speed of the run |
| steps               | Number  | Number of steps in the run |
| timeStamp           | Number  | Timestamp of the run |
| totalDistance       | Number  | Total distance covered in the run |
| totalLitter         | Number  | Total litter count |
| userID              | String  | ID of the user |
| userName            | String  | Name of the user |
| userTeam            | String  | Name of the user's team |

### Example Requests

Curl Request:
```bash
curl --location 'https://qr-dev.rubbish.love/public/api/v1/runs/<teamId>/movement/<runId>' \
--header 'Authorization: Bearer <token>'
```

JavaScript Fetch Request:
```bash
var myHeaders = new Headers();
myHeaders.append("Authorization", "Bearer <token>");

var requestOptions = {
  method: 'GET',
  headers: myHeaders,
  redirect: 'follow'
};

fetch("https://qr-dev.rubbish.love/public/api/v1/runs/<teamId>/movement/<runId>", requestOptions)
  .then(response => response.text())
  .then(result => console.log(result))
  .catch(error => console.log('error', error));
```
